# SplitSend

ブラウザだけで、大きなファイルを指定サイズ以下に**分割**し、受信側で元のファイルへ**統合（復元）**するためのWebツールです。

ファイルはサーバーへアップロードされず、分割・検証・復元は利用者のブラウザ内で処理されます。Pythonや専用アプリのインストールは不要です。

## 公開ページ

| 操作 | URL |
|---|---|
| ファイルを分割する | https://temesotejam.github.io/splitsend-web/#split |
| 分割ファイルを統合・復元する | https://temesotejam.github.io/splitsend-web/#restore |

> [!IMPORTANT]
> 分割と復元には、デスクトップ版の **Google Chrome** または **Microsoft Edge** を推奨します。ブラウザのフォルダー選択・ファイル保存機能を利用するため、GitHub PagesのHTTPSページから開いてください。

---

## SplitSendでできること

- 任意形式の単一ファイルを指定サイズ以下に分割
- Discord向けの `9.5 MB` プリセット
- `25 MB`、`49 MB`、`95 MB`、任意サイズでの分割
- 分割結果を専用サブフォルダーへまとめて保存
- すべての処理をブラウザ内で実行
- 欠けているパーツの検出
- 重複パーツの検出
- ファイルサイズの不一致検出
- 各パーツのSHA-256検証
- 復元後のファイル全体のSHA-256検証
- 元ファイルと1バイト単位で一致したことの確認

## 対応するファイル形式

ファイルの内容を解析せず、バイト列として分割するため、基本的にあらゆる**単一ファイル**を扱えます。

例：

- ZIP、7z、RAR、tar.gz
- PDF、DOCX、XLSX、PPTX、TXT
- PNG、JPEG、WebP、TIFF、RAW
- MP4、MOV、AVI、MKV
- MP3、WAV、FLAC、AAC
- EXE、MSI、APK
- ISO、IMG
- CSV、BIN、ログファイル
- STL、STEP、CADデータ
- GGUF、ONNX、safetensors
- 拡張子のないファイル

フォルダーは直接分割できません。フォルダーを送りたい場合は、先に通常のZIPファイルへまとめてからSplitSendで分割してください。

# ファイルを分割する方法

分割ページ：

https://temesotejam.github.io/splitsend-web/#split

## 1. 分割するファイルを選ぶ

ページの「ファイルをここへドロップ」の領域へファイルをドラッグ＆ドロップします。

ドラッグ＆ドロップしない場合は、領域をクリックしてファイルを選択できます。

選択後、次の情報が表示されます。

- ファイル名
- ファイルサイズ
- ファイル形式
- 最大パーツサイズ
- 生成されるパーツ数
- 最後のパーツのサイズ

## 2. 1パーツの最大サイズを選ぶ

「1パーツの最大サイズ」から設定を選択します。

| 設定 | 主な用途 |
|---|---|
| Discord向け 9.5 MB | Discordの10 MB制限に余裕を持たせたい場合 |
| 25 MB | メールや各種アップロードサービス向け |
| 49 MB | 50 MB制限に余裕を持たせたい場合 |
| 95 MB | 100 MB制限に余裕を持たせたい場合 |
| カスタム | 任意のサイズを指定したい場合 |

カスタムでは、`MB` または `MiB` を選択できます。

- `1 MB = 1,000,000 bytes`
- `1 MiB = 1,048,576 bytes`

送信先の上限ぎりぎりではなく、少し小さい値を指定することを推奨します。

## 3. 分割予定を確認する

ファイルと分割サイズを指定すると、次の内容が表示されます。

- 最大パーツサイズ
- 生成パーツ数
- 最後のパーツサイズ

パーツ数が多すぎる場合は、分割サイズを大きくするか、別の送信手段も検討してください。

## 4. 保存先を選ぶ

「保存先を選択して分割」を押します。

ブラウザのフォルダー選択画面が開くので、親フォルダーを選択します。通常は「ダウンロード」フォルダーなどを選べば問題ありません。

選択した親フォルダーの直下へ大量のファイルを直接保存するのではなく、SplitSendが専用サブフォルダーを自動作成します。

例：

```text
Downloads/
└─ SplitSend_example_A1B2C3D4/
   ├─ example_A1B2C3D4.part001
   ├─ example_A1B2C3D4.part002
   ├─ example_A1B2C3D4.part003
   ├─ example_A1B2C3D4.splitsend.json
   └─ 復元方法.txt
```

そのため、分割ファイルがダウンロードフォルダーやデスクトップへばらばらに散らばることはありません。

## 5. 分割完了まで待つ

処理中は、次の情報が表示されます。

- 現在処理しているパーツ
- 進捗率
- 処理済みデータ量
- 現在の出力ファイル名

処理が終わるまで、ページを閉じたり再読み込みしたりしないでください。

途中で停止する場合は「キャンセル」を押します。

## 6. 完了結果を確認する

正常に完了すると、次の内容が表示されます。

- 保存フォルダー名
- 生成されたパーツ数
- 元ファイルのSHA-256

SHA-256は、復元したファイルが元ファイルと完全に一致しているか確認するための値です。

# どのファイルを相手へ送るか

## 復元に必須

次のファイルは**すべて必要**です。

```text
example_A1B2C3D4.part001
example_A1B2C3D4.part002
example_A1B2C3D4.part003
example_A1B2C3D4.splitsend.json
```

- すべての `.partXXX` ファイル
- `.splitsend.json` ファイル

## 任意

```text
復元方法.txt
```

`復元方法.txt` は受信者向けの説明ファイルです。復元処理そのものには不要ですが、一緒に送ると親切です。

## 送信時の注意

- `.part` ファイルを1個でも送り忘れると復元できません。
- `.splitsend.json` を送り忘れると復元できません。
- ファイル名を変更しないでください。
- ファイル内容を編集しないでください。
- Discordへ送る順番は問いません。
- 複数メッセージに分けて送っても問題ありません。
- 受信者がすべてのファイルをダウンロードできるようにしてください。

> [!TIP]
> 専用フォルダーを開いて、必要なファイルをすべて選択し、そのままDiscordへドラッグすると送り忘れを減らせます。

# 分割ファイルを統合・復元する方法

復元ページ：

https://temesotejam.github.io/splitsend-web/#restore

## 1. 必要なファイルをすべてダウンロードする

送信者から受け取った次のファイルを、すべてPCへ保存します。

- 全 `.partXXX` ファイル
- `.splitsend.json`

`復元方法.txt` はなくても復元できます。

ダウンロードしたファイルが同じフォルダーに入っていなくても選択できますが、あらかじめ1つのフォルダーへまとめておくと作業しやすくなります。

## 2. 復元ページを開く

次のページを開きます。

https://temesotejam.github.io/splitsend-web/#restore

画面上部のタブを使う場合は「復元」を選択します。

## 3. 分割ファイル一式を選択する

`.part` ファイルと `.splitsend.json` をすべて選択し、「分割ファイル一式をここへドロップ」へドラッグ＆ドロップします。

領域をクリックして、複数ファイルをまとめて選択することもできます。

ファイル選択画面では、`Ctrl` キーや `Shift` キー、または `Ctrl + A` を使って必要なファイルをまとめて選択してください。

## 4. 検出結果を確認する

SplitSendがパッケージIDを読み取り、必要なファイルがそろっているか確認します。

正常な場合は、次のように表示されます。

```text
必要なファイルがそろっています。
復元時にSHA-256を検証します。
```

問題がある場合は、具体的なファイル名とともに次のような内容が表示されます。

- 不足
- 同名パーツ重複
- サイズ不一致
- 復元情報エラー

複数の分割セットを同時に選択した場合は、検出したパッケージが複数表示されます。復元したいパッケージを選択してください。

## 5. 復元先を選ぶ

「復元先を選択して実行」を押します。

ブラウザの保存画面が開きます。元のファイル名が候補として表示されるので、保存先とファイル名を確認して保存します。

同名のファイルがすでにある場合は、上書きするか、別のファイル名で保存してください。

## 6. 検証と復元を待つ

復元中は、各パーツについて次を確認します。

1. ファイル名
2. ファイルサイズ
3. 各パーツのSHA-256
4. パーツの順番
5. 復元後の合計サイズ
6. 復元ファイル全体のSHA-256

処理中はページを閉じたり、再読み込みしたりしないでください。

## 7. 完了表示を確認する

正常に復元されると、次の情報が表示されます。

- 復元ファイル名
- 復元ファイルサイズ
- SHA-256
- 「元ファイルと完全に一致しています」という確認

この表示が出れば、元ファイルと復元ファイルは1バイト単位で一致しています。

# エラーが表示された場合

## 「必要なパーツが不足しています」

原因：

- `.part` ファイルの一部をダウンロードしていない
- 一部のファイルを選択し忘れた
- ファイル名が変更されている

対処：

1. `.splitsend.json` に対応する全パーツを確認します。
2. Discordなどから不足ファイルを再度ダウンロードします。
3. すべてのパーツとJSONをまとめて選択し直します。

## 「同名パーツ重複」

原因：

- 同じパーツを2回選択している
- ダウンロード時に同じファイルが複製されている

対処：

- 同じ番号のパーツを1個だけ残して選択し直してください。

## 「サイズ不一致」

原因：

- ダウンロードが途中で止まった
- ファイルが壊れている
- ファイル内容が変更された

対処：

- 表示されたパーツを削除し、送信元から再ダウンロードしてください。

## 「SHA-256が一致しません」

原因：

- パーツが破損している
- 別の分割セットのパーツが混ざっている
- ファイル内容が変更されている

対処：

1. エラーで示されたパーツを再ダウンロードします。
2. ファイル名末尾のパッケージIDがそろっているか確認します。
3. すべてのファイルを選択し直します。

## ボタンを押してもフォルダー選択画面が出ない

確認事項：

- GitHub PagesのURLを開いているか
- `file:///.../index.html` のようにPC上のHTMLを直接開いていないか
- デスクトップ版ChromeまたはEdgeを使用しているか
- ブラウザがフォルダーやファイルへのアクセスを拒否していないか
- プライベートブラウジングや組織のセキュリティ設定で機能が制限されていないか

一度ページを再読み込みし、アクセス許可を承認してから再度実行してください。

## 保存容量が不足する

分割では、出力されるパーツの合計サイズは元ファイルとほぼ同じです。復元でも、元ファイルと同程度の空き容量が必要です。

不要なファイルを削除するか、空き容量のある別ドライブを保存先にしてください。

# 重要な注意事項

## 分割は圧縮ではない

SplitSendはファイルを小さなパーツへ分けますが、データ量そのものを大幅に減らすものではありません。

```text
元ファイル：100 MB
分割結果：9.5 MB前後のパーツが複数
合計：約100 MB
```

ファイルサイズ自体を減らしたい場合は、分割前にZIP圧縮、画像圧縮、動画変換などを行ってください。

## 分割は暗号化ではない

すべてのパーツと `.splitsend.json` を持っている人は、元ファイルを復元できます。

機密情報を送る場合は、SplitSendで分割する前に、パスワード付きZIPや信頼できる暗号化ツールなどで別途暗号化してください。

## すべてのパーツが必要

パーツが1個でも欠けると、元ファイルを完全には復元できません。

送信者は送信前、受信者は復元前に、パーツ番号が連続していることを確認してください。

例：

```text
part001
part002
part003
part004
```

`part003` がない場合は復元できません。

## ファイル名を変えない

復元情報には各パーツの正確なファイル名が記録されています。OSやブラウザが自動で `(1)` などを付けた場合は、重複ダウンロードがないか確認してください。

# よくある質問

## ZIP、動画、PDF、EXEも分割できますか？

はい。単一ファイルであれば、内容や拡張子に関係なく分割できます。

## フォルダーをそのまま分割できますか？

現在はできません。先にフォルダーをZIPファイルへまとめてください。

## Pythonは必要ですか？

不要です。分割も復元もWebページ上で行います。

## 分割パーツを送る順番は決まっていますか？

送信順は問いません。復元時は `.splitsend.json` の情報に従って正しい順番で結合します。

## `.splitsend.json` をなくしても復元できますか？

このWeb版では復元できません。パーツと一緒に必ず保管・送信してください。

## `復元方法.txt` は必須ですか？

必須ではありません。受信者向けの説明用ファイルです。

## スマートフォンで使えますか？

ブラウザのファイル保存機能やメモリ制限が異なるため、大容量ファイルでは推奨しません。デスクトップ版ChromeまたはEdgeを使用してください。

## ファイルはサーバーへ送信されますか？

送信されません。ファイルの読み込み、分割、ハッシュ計算、復元、保存はブラウザ内で処理されます。

## 何GBまで使えますか？

固定の保証値はありません。実際の上限は、ブラウザ、OS、ストレージ空き容量、ファイルシステムによって変わります。最初は小さなファイルで動作確認してから、大容量ファイルを扱ってください。

# プライバシーと安全性

SplitSendは次の方針で動作します。

- ファイルを外部サーバーへアップロードしない
- ファイル内容を外部へ送信しない
- アナリティクスを使用しない
- 広告を使用しない
- 外部CDNや外部JavaScriptを使用しない
- 分割・復元処理をブラウザ内で行う
- 各パーツと復元結果をSHA-256で検証する

ただし、分割されたファイル自体は暗号化されません。
