# Security

## 脅威モデル

SplitSendは、利用者が自分で選んだローカルファイルを分割・復元するツールです。マニフェストやパーツは第三者から受け取る可能性があるため、信頼しない入力として扱います。

## 実装済み対策

- マニフェスト構造の厳格な検証
- 未対応バージョンの拒否
- パス区切り文字と制御文字の拒否
- Windows禁止文字と予約名の無害化
- パーツ名の重複検出
- パーツ数の上限
- マニフェストサイズの上限
- 各パーツのサイズ検証
- 各パーツのSHA-256検証
- 復元全体のSHA-256検証
- 書き込みストリームを閉じる前の完全性確認
- エラー時のストリーム中止
- 分割失敗時の専用フォルダー削除
- Content Security Policyによる外部通信禁止
- `eval()`、動的外部コード、外部依存関係なし

## 非対応事項

SplitSendは暗号化、アクセス制御、マルウェアスキャンを行いません。実行ファイルなどを復元した場合、その安全性は元ファイルと同じです。

## 脆弱性報告

公開リポジトリのSecurity AdvisoryまたはIssueを利用してください。機密性のある報告は、公開Issueへ詳細を書かず、Security Advisoryを推奨します。
