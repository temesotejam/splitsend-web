# Test Plan

## 自動テスト

`npm run check` で実行します。

### SHA-256

- 空入力
- `abc`
- 長い既知入力
- 異なるチャンク境界での逐次更新
- Node.js標準暗号実装との比較

### 分割・復元

- 複数パーツへの分割
- 最終パーツだけ小さいケース
- 0バイトファイル
- 分割と異なるチャンクサイズで復元
- 復元後のバイト完全一致
- パーツ1バイト改変時の拒否
- エラー時の出力中止

### マニフェスト

- 正常形式
- オフセット不連続
- 危険なパス
- 重複名
- 合計サイズ不一致
- Windows禁止ファイル名

### 静的サイト

- 必須ファイルの存在
- 外部アセット参照がないこと
- Fetch、XMLHttpRequest、WebSocketを使用していないこと
- `connect-src 'none'` があること

## 手動テスト

### サイズ境界

- 0 B
- 1 B
- 分割サイズ未満
- 分割サイズと一致
- 分割サイズ + 1 B
- 30 MB
- 100 MB
- 500 MB
- 1 GB
- 4 GB超（十分な空き容量がある環境）

### ファイル形式

- TXT
- CSV
- PDF
- ZIP
- JPEG
- MP4
- EXE
- BIN
- 拡張子なし

### 操作異常

- 保存先選択をキャンセル
- 分割途中でキャンセル
- 復元途中でキャンセル
- partを1個削除
- partを別パッケージと交換
- partを1バイト変更
- JSONを改変
- 同名パーツを複数選択
- 保存先の空き容量不足
- 同名出力ファイルの上書き確認

## 完成基準

- すべての自動テストが成功
- Chromeで分割・復元が成功
- SHA-256が元ファイルと一致
- パーツが専用フォルダー内に出力される
- 破損時に完成ファイルが確定しない
